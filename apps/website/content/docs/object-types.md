Most notes are just notes. But some come in sets — every meeting, every book, every interview — and those benefit from a shape. An object type is that shape: a small schema you define once and attach to any note or task.

## Define a type

Go to **Settings → Objects** and click **New type**. Give it a name, choose whether it applies to notes, tasks, or both, pick an icon and a colour, then add fields.

![The object type editor, defining a Meeting type with three fields](/docs/object-type.png)

Fields can be text, number, date, checkbox, URL, a single or multi-select from options you list, or a reference to another note or task. Mark a field required and Companion will hold you to it.

## Use it

Open a note and click **Add type** beside the title. Its fields appear in a metadata panel next to the note — fill them in there while the body of the note stays free-form prose.

![A note typed as a Meeting, with its fields filled in](/docs/note-metadata.png)

## Why bother

Three things fall out of it. Typed items are coloured and marked by their icon in [the graph](/docs/using-the-graph), so a whole category is visible at once. Reference fields create real links, which means your structure shows up in the graph too. And the [AI](/docs/chatting-with-companion) can query these fields — "which meetings decided something about pricing?" — because it can read the schema, not just the prose.

> **Tip:** Start without types. Add one when you notice you've written the same three lines at the top of five notes in a row.
