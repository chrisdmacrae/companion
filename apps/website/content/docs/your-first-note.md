Notes are where your ideas live in Companion. They're markdown you can write freely, link to anything else in your workspace, and give structure to when you're ready. Here's how to create your first one.

## Create a note

Open **Notes** in the sidebar and click the **＋** in the top right. A new note opens straight into the editor.

Give it a title on the first line — that title is how the note is referenced everywhere else, so it's worth a real one. Then keep typing. Notes save as you go; there's no save button to hunt for.

![The note editor with a title, a heading, and a bulleted list](/docs/note-editor.png)

Formatting happens as you type: `## ` starts a heading, `- ` a bullet, `1. ` a numbered list, `> ` a quote, and `**bold**` or `*italic*` do what you'd expect. A formatting bar floats at the bottom of the editor while you write, with buttons for bold, italic, strikethrough, code, code blocks, quotes, lists, tables, and file attachments.

## Link to anything

Companion is built around connections. Type `[[` anywhere in a note and start searching — you'll see matching notes *and* tasks, each labelled with its kind. Pick one and Companion inserts a live chip showing the item's title, and for tasks its due date.

![Typing two square brackets opens a search across notes and tasks](/docs/note-wikilink.png)

Links run both ways. The icon in the top right of any note or task opens a small graph of everything it touches — including the notes that link *to* it.

![A note's graph panel showing the items it connects to](/docs/note-graph.png)

These same links power [the graph](/docs/using-the-graph), a map of your whole workspace.

## Attach files

Paste a file, drop it onto the editor, or use **Attach file** in the formatting bar. Images preview inline, audio gets a player, and anything else becomes a file pill you can click. Attachments are first-class: they appear in the graph alongside your notes and tasks.

## Give a note structure

Notes can also be *typed*. Click **Add type** beside the title to attach an object type — say **Meeting**, with fields for attendees, the decision, and a follow-up date — and those fields appear in a metadata panel beside the note.

![A note typed as a Meeting, with its fields in the metadata panel](/docs/note-metadata.png)

It's optional, and most notes never need it. But when you want a set of notes to stay consistent — meetings, interviews, reading notes — [object types](/docs/object-types) are how you get there.

> **Tip:** Not sure where a note belongs yet? Leave it unsorted. You can drop it into a [project](/docs/your-first-project) later with the folder icon in the toolbar.
